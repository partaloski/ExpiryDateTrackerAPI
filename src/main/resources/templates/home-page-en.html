Expiry Date Tracker API

What is the Expiry Date Tracker API?

An API made in Java using Spring Boot and Rest API controllers, to help an external website or a mobile application communicate with a database without the developer worrying too much, a quick summary of all of the functions available can be found down below, tap on the big blue buttons to get information about certain parts of the API.

Products

GET

/api/products/

Retrieves a list of all products.

Expects no variables.

POST

/api/products/add

Used to add a product into the database.

Expects 3 Request Variables:
id (String), name (String), manufacturer_id (Integer).

DELETE

/api/products/delete/{ID}

Deletes the product which can be identified with the [ID] attribute.

Expects 1 Path Variable:
id (String)

Manufacturers

GET

/api/manufacturers/

Retrieves a list of all manufacturers.

Expects no variables.

POST

/api/manufacturers/add

Used to add a manufacturer into the database.

Expects 1 Request Variable:
name (String).

DELETE

/api/manufacturers/delete/{ID}

Deletes the manufacturer which can be identified with the [ID] attribute.

Expects 1 Path Variable:
id (Integer)

Users

POST

/api/users/login

Used to log in as a certain user, returns an authentication code, used to then authenticate the user and their actions.

Expects 2 Request Variables:
username (String), password (String).

POST

/api/users/register

Used to register a new user into the database.

Expects 6 Request Variables:
username (String), password (String), confirmPassword (String), name (String), surname (String), email (String).

GET

/api/users/{username}/wishlist

Used to get all wishlist items for a certain user, if the authentication code matches.

Expects 1 Path Variable and 1 Request Variable:
username (String), auth_code(String).

GET

/api/users/{username}/expiryList

Used to get all expiry items for a certain user, if the authentication code matches.

Expects 1 Path Variable and 1 Request Variable:
username (String), auth_code(String).

Products-Users

POST

/api/productsUsers/add/e

Used to add a relation User-Product, which marks a date when a product expires, if the authentication code matches.

Expects 4 Request variables:
username(String), productId(String),
expiryDate(LocalDate, "YYYY-MM-DD"), auth_code(String).

DELETE

/api/productsUsers/delete/e

Used to delete a certain expiry from the database, if the authentication code matches.

Expects 2 Request Variables:
id (Integer), auth_code(String).

DELETE

/api/productsUsers/
{username}/clear/e

Clears the user with the username variable's expiry list, if the authentication code matches.

Expects 1 Path Variable and 1 Request Variable:
username (String), auth_code(String).

POST

/api/productsUsers/add/w

Used to add a relation User-Product, which marks a date when a user wants to buy (wishlists) a product with a given quantity, if the authentication code matches.

Expects 4 Request variables:
username(String), productId(String), quantity(Integer)
, auth_code(String).

DELETE

/api/productsUsers/delete/w

Used to delete a certain wishlist item from the database, if the authentication code matches.

Expects 3 Request Variables:
username (String), productId (String), auth_code(String).

DELETE

/api/productsUsers/
{username}/clear/w

Clears the user with the username variable's wishlist, if the authentication code matches.

Expects 1 Path Variable and 1 Request Variable:
username (String), auth_code(String).
